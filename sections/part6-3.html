📊 第一题：关键路径法 (CPM)

背景：一个软件开发项目包含 A, B, C, D, E, F, G 七项活动

活动 紧前活动 持续时间（天）
AStart4
BStart3
CA5
DA2
EB, D6
FC, E4
GE3

要求：

  1. 绘制此项目的网络图
  2. 计算每项活动的最早开始时间(ES)、最早完成时间(EF)、最晚开始时间(LS)、最晚完成时间(LF)和总浮动时间(TF)
  3. 找出该项目的所有关键路径及其总工期

详细解答过程

步骤1：正向传递 (Forward Pass) - 计算ES和EF

A: ES=0, EF=0+4=4

B: ES=0, EF=0+3=3

C: ES=4, EF=4+5=9

D: ES=4, EF=4+2=6

E: ES=max(3,6)=6, EF=6+6=12

F: ES=max(9,12)=12, EF=12+4=16

G: ES=12, EF=12+3=15

项目工期 = max(16,15) = 16天

步骤2：逆向传递 (Backward Pass) - 计算LS和LF

F: LF=16, LS=16-4=12

G: LF=16, LS=16-3=13

E: LF=min(12,13)=12, LS=12-6=6

C: LF=12, LS=12-5=7

D: LF=6, LS=6-2=4

B: LF=6, LS=6-3=3

A: LF=min(7,4)=4, LS=4-4=0

步骤3：活动时间参数汇总表
活动持续时间ESEFLSLFTF
A404040
B303363
C5497123
D246460
E66126120
F4121612160
G3121513161
最终答案：
  • 关键路径：A → D → E → F (TF=0的活动)
  • 项目总工期：16天
  • 关键活动：A、D、E、F

💰 第二题：挣值管理 (EVM) 基础分析

背景：移动应用开发项目第4个月末数据

项目总预算 (BAC)

80万元

计划工期10个月

第4个月末数据

PV: 30万元

EV: 24万元

AC: 28万元

要求：

  1. 计算项目的成本偏差(CV)和进度偏差(SV)
  2. 计算项目的成本绩效指数(CPI)和进度绩效指数(SPI)
  3. 全面分析项目当前状态（成本和进度两个角度）

详细解答过程

已知数据：

BAC = 80万元，PV = 30万元，EV = 24万元，AC = 28万元

1. 偏差计算：

成本偏差 (CV) = EV - AC = 24 - 28 = -4万元

进度偏差 (SV) = EV - PV = 24 - 30 = -6万元

负值表示成本超支且进度滞后

2. 绩效指数计算：

成本绩效指数 (CPI) = EV / AC = 24 / 28 ≈ 0.857

进度绩效指数 (SPI) = EV / PV = 24 / 30 = 0.8

小于1表示效率低于预期

3. 项目状态分析：

成本方面：

  • CV = -4万元，CPI = 0.857 < 1，项目成本超支
  • 每花费1元只完成0.857元预算的工作，资金使用效率低

进度方面：

  • SV = -6万元，SPI = 0.8 < 1，项目进度落后
  • 实际完成工作量只达到计划的80%

结论：项目处于成本超支且进度滞后的"双重困境"，需立即采取纠正措施！

🔮 第三题：挣值管理 (EVM) 预测

背景：沿用第二题数据进行未来成本趋势预测

已知数据：

BAC = 80万元，EV = 24万元，AC = 28万元，CPI = 0.857

要求：

  1. 假设项目未来绩效保持当前水平，计算完工估算(EAC)
  2. 计算完工尚需估算(ETC)
  3. 计算完工偏差(VAC)
  4. 为按原预算完成项目，剩余工作需要的成本绩效指数(TCPI)是多少？现实吗？

详细解答过程

1. 完工估算 (EAC)：

EAC = BAC / CPI = 80 / 0.857 ≈ 93.35万元

按当前趋势，项目完成时预计总花费93.35万元

2. 完工尚需估算 (ETC)：

ETC = EAC - AC = 93.35 - 28 = 65.35万元

完成剩余工作预计还需要65.35万元

3. 完工偏差 (VAC)：

VAC = BAC - EAC = 80 - 93.35 = -13.35万元

预计项目最终将超出原预算13.35万元

4. 完工尚需绩效指数 (TCPI)：

TCPI = (BAC - EV) / (BAC - AC) = (80 - 24) / (80 - 28) = 56 / 52 ≈ 1.077

剩余工作每花费1元必须产生1.077元价值

目标现实性分析：

这个目标极具挑战性：

  • 当前CPI仅为0.857（85.7%效率）
  • 要求未来提升到107.7%效率
  • 需要将成本效率提升25.6%
  • 虽然困难但并非完全不可能

建议：项目经理需立即实施严格的成本控制措施

⏰ 第四题：三点估算 (PERT) 与期望工期

背景：关键路径上三项连续活动 (X, Y, Z) 的工期估算

活动 最乐观时间(TO) 最可能时间(TM) 最悲观时间(TP)
X8天10天18天
Y5天7天15天
Z6天9天12天

要求：

  1. 使用三点估算法(PERT)计算每项活动的期望历时(TE)
  2. 计算这三项活动构成的路径的总期望工期

详细解答过程

PERT三点估算公式：

TE = (TO + 4×TM + TP) / 6

各活动期望历时计算：

活动X：

TE(X) = (8 + 4×10 + 18) / 6 = (8 + 40 + 18) / 6 = 66 / 6 = 11天

活动Y：

TE(Y) = (5 + 4×7 + 15) / 6 = (5 + 28 + 15) / 6 = 48 / 6 = 8天

活动Z：

TE(Z) = (6 + 4×9 + 12) / 6 = (6 + 36 + 12) / 6 = 54 / 6 = 9天

最终答案：

各活动期望历时：

  • 活动X：11天
  • 活动Y：8天
  • 活动Z：9天

总期望工期 = 11 + 8 + 9 = 28天

🗣️ 第五题：沟通渠道计算

背景：敏捷开发团队人员调整

初始团队

7名成员

包括Scrum Master和产品负责人

调整后

+2名开发人员

从其他项目组临时调入

要求：

  1. 计算团队增加成员之前的沟通渠道数量
  2. 计算团队增加成员之后的沟通渠道数量
  3. 从项目管理角度，说明沟通渠道急剧增加可能带来的管理挑战

详细解答过程

沟通渠道计算公式：

沟通渠道数 = N × (N - 1) / 2

其中N为团队成员数量

1. 增加成员前的沟通渠道：

N = 7（成员数）

沟通渠道数 = 7 × (7 - 1) / 2 = 7 × 6 / 2 = 21条

2. 增加成员后的沟通渠道：

N = 7 + 2 = 9（成员数）

沟通渠道数 = 9 × (9 - 1) / 2 = 9 × 8 / 2 = 36条

3. 变化分析：

人员增加：7 → 9（增加28.6%）

沟通渠道：21 → 36（增加71.4%）

沟通复杂度的增长远超人员增长比例！

管理挑战分析：

1. 沟通开销急剧增加

  • 沟通时间和精力不成比例地大幅增加
  • 用于实际开发的时间可能反而减少

2. 信息传递失真风险增高

  • 沟通路径变长变多，易出现误解和遗漏
  • 决策失误风险增加

3. 新成员融入困难

  • 需要时间理解现有沟通模式和团队文化
  • 可能暂时打乱原有沟通节奏
  • 管理复杂度显著上升

常用公式参考手册

CPM关键路径法：

  • 总浮动时间 TF = LS - ES = LF - EF
  • 自由浮动时间 = ES(后续) - EF(当前)
  • 关键路径：TF = 0 的活动序列

EVM挣值管理：

  • CV = EV - AC，CPI = EV / AC
  • SV = EV - PV，SPI = EV / PV
  • EAC = BAC / CPI，VAC = BAC - EAC
  • TCPI = (BAC - EV) / (BAC - AC)

其他重要公式：

  • PERT: TE = (TO + 4×TM + TP) / 6
  • 沟通渠道 = N × (N - 1) / 2
  • ETC = EAC - AC
